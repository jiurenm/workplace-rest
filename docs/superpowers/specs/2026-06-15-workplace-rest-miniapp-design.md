# 工位小憩小程序骨架设计

## 背景

工位小憩面向久坐办公人群，MVP 需要先验证喝水、护眼、站立提醒、数据统计、设置和微信订阅消息是否能支撑长期使用。当前阶段只搭建微信原生小程序骨架，不实现具体页面 UI。

## 技术选型

- 前端使用微信原生小程序 + TypeScript。
- 后端优先使用微信云开发，包含云函数、云数据库和云存储。
- 前端通过 `services/` 访问云开发能力，页面不直接散落 `wx.cloud` 调用，便于未来迁移到 Cloudflare Workers 或其他后端。

## 范围

本次创建：

- 小程序入口配置：`app.ts`、`app.json`、`app.wxss`。
- 四个主 tab 页面：首页、数据、设置、我的。
- 业务模型：提醒任务、打卡记录、用户设置、统计数据。
- 服务层：云开发初始化、提醒、打卡、设置、订阅消息。
- 占位组件：倒计时环、任务概览、动作指导弹层。
- 云函数占位：发送订阅提醒、统计聚合。
- 开发配置：`project.config.json`、`tsconfig.json`、`package.json`、`sitemap.json`。

不包含：

- 具体 UI 还原。
- 真实订阅消息模板 ID。
- 云数据库集合创建脚本。
- 后台定时触发器部署配置。
- 成就系统、分享海报、如厕和刷牙完整实现。

## 页面结构

- `pages/home`：首页入口，占位展示当前页面名称，后续接入倒计时环、快捷打卡和今日概览。
- `pages/stats`：数据页入口，后续接入趋势图、打卡日历和今日统计。
- `pages/settings`：设置页入口，后续接入工作时间、提醒频率、开关和专注模式。
- `pages/profile`：我的入口，后续接入个人信息、声音震动、反馈和关于我们。

## 数据模型

核心任务类型包括：

- `water`：喝水。
- `eye`：护眼。
- `stand`：站立。
- `toilet`：如厕，MVP 暂缓。
- `brush`：刷牙，MVP 暂缓。

模型拆分为：

- `ReminderTask`：提醒任务配置和下一次提醒时间。
- `CheckinRecord`：用户打卡记录。
- `UserSettings`：用户设置。
- `DailyStats`：每日统计。

## 云开发边界

`services/cloud.ts` 负责云开发初始化和统一调用。业务服务只依赖该封装：

- `reminderService`：读取提醒任务、暂停/恢复、计算下一次提醒。
- `checkinService`：提前打卡和今日统计。
- `settingsService`：读取/保存设置。
- `subscriptionService`：订阅消息授权和发送请求入口。

## 风险和后续

- 微信小程序不能长期后台计时，后续提醒实现应以订阅消息和云函数定时扫描为核心。
- Cloudflare 后端暂不作为 MVP 默认方案，但服务层会保留迁移边界。
- 当前目录不是 git 仓库，本设计文档不会提交。
